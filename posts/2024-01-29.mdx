> GraphQL이란

<aside>
💡 GraphQL은 쿼리언어이자 서버측 런타임

</aside>

그래프 큐엘은 데이터를 가져오는데 쿼리를 이용하도록 합니다.

> runtime

graphQL api는 서버에서 runtime에 실행된다.

데이터의 출처에 구애받지 않고 runtime에 데이터 패칭이 가능하다.

> GraphQL 구현 순서

1. GraphQL schema design
2. resolver ↔ data source connection
3. GraphQL query fetching

> GraphQL 의 특징

1. 성능
   1. 오버패칭 방지
   2. 언더패칭 방지: rest API의 경우 서로 다른 schema 또는 data source의 경우 여러번의 api콜로 각각 언더패칭된 데이터를 front-side에서 조합해야 하는 경우가 있다. graphQL은 이런 문제를 해결해준다.
2. 개발자 편의성
   1. 계층적 스키마 구조: 그래프 구조와 선언적 구조를 통해 쿼리를 직관적으로 이해할 수 있다.
   2. 타입 안정성: graphQL은 그 자체로 스키마 구조를 가지므로 프론트 사이드와 서버 사이드의 타입이 상호 동일해야만 통신이 가능하므로 그 자체로 타입안정성을 보장한다.
3. 구조
   1. 클라이언트 개발의 서버 의존성 분리: restAPI처럼 서버사이드에서 원하는 API를 개발해 줄 때까지 기다려야 하는 업무의존성을 없앨 수 있다.
   2. 단일 정보출처: 프론트 사이드에선 데이터들 마다 API를 언더패칭하여 조합하던 restAPI와 달리 조합이 되어 불려오도록 하는 단일 출처만 알면된다.
4. 훌륭한 커뮤니티
   1. 훌륭한 도구들: apollo client, apollo studio 등 다양한 도구들이 완비되어있다.
   2. 광범위한 이용: 여러 사이트 들에서 graphQL을 이용중이다.
